# Hidden

When using Quick Models or Custom Models, Larawiz will automatically set as hidden any attribute that contains:

  • password,
  • secret,
  • private,
  • hidden,
  • or is a rememberToken.



     
     
     




     

    models:
  User:
    name: string
    email: string
    password: string
    rememberToken: ~
  Post:
    columns: 
      title: string
      body: text
      private_notes: json



     






     



    class User extends Authenticatable
{
    protected $hidden = [
        'password', 'remember_token'
    ];
}

class Post extends Model
{
    protected $hidden = [
        'private_notes'
    ];
}

    If you're using a Custom Model, you have full control on what is hidden or not by using the hidden key inside the model declaration.








       
       
       

      models:
  Post:
    columns: 
      title: string
      body: text
      private_notes: json
      editor_notes: json
    hidden:
      - author_notes
      - private_notes



       



      class Post extends Model
{
    protected $hidden = [
        'author_notes', 'private_notes',
    ];
}

      Don't bother me

      Larawiz won't bother checking if the hidden properties doesn't exist in the column definitions.

      Alternatively, you can disable any hidden guessing by issuing hidden with false. No hidden properties will be added to the model.








       

      models:
  Post:
    columns: 
      title: string
      body: text
      private_notes: json
      editor_notes: json
    hidden: false
