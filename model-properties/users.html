# Users

You may want to create an Authenticable (opens new window) and Authorizable (opens new window) user, like the one shipping by default with Laravel.

If you're using a Quick Model, issuing a password, rememberToken, or both, will instruct Larawiz to treat the model as an User Model.





     
     


    models:

  Admin:
    name: string unique
    password: string
    rememberToken: ~
    age: unsignedSmallInteger default:18

    <?php

namespace App\Models;

use Illuminate\Contracts\Auth\MustVerifyEmail;
use Illuminate\Foundation\Auth\User as Authenticatable;
use Illuminate\Notifications\Notifiable;

class Admin extends Authenticatable
{
    use Notifiable;
}

    If you're using Custom Models, you will need to change the type of the model by just setting the type key to user, even if there is already a password or rememberToken column.












     

    models:

  Admin:
    columns:
      id: ~
      name: string unique
      password: string
      rememberToken: ~
      age: unsignedSmallInteger default:18
      timestamps: ~

    type: user

    TIP

    If you set an User model with password, Larawiz will add a mutator (opens new window) to automatically encrypt the password into the model attributes, for free.

    class Admin extends Authenticatable
{
    // ...

    public function setPasswordAttribute($password)
    {
        $this->attributes['password'] = app('hash')->make($password);
    }
}
